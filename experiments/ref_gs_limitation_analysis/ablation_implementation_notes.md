# Ablation Implementation Notes

Date: 2026-07-06

## Existing config-only ablations

These can be launched without changing model code:

- `--run_dim`: changes the MLP hidden width used by `GaussianModel.light_mlp`.
- `--gsrgb_loss`: available in `train-NeRF.py` as an extra stock RGB supervision term.
- `--roughness_lr`: can be set to `0` to freeze roughness after initialization.
- `--feature_lr`, `--encoding_lr`, `--mlp_lr`: can reduce learning capacity or isolate training sensitivity.
- `--rand_init`: changes Sph-Mip encoding initialization through `GaussianModel`.
- `--lambda_dssim`, `--lambda_dist`, `--lambda_normal`: affect loss weighting but do not disable the factorization itself.

## no-Sph-Mip

Current status: not implemented as a CLI flag.

Relevant code:

- `scene/gaussian_model.py`: `SphMipEncoding(...)` is created as `self.dir_encoding`.
- `gaussian_renderer/__init__.py`: `pc.dir_encoding(...)` is called in `render`, `render_nerf`, and `render_real` before `light_mlp`.

Minimal implementation path:

1. Add a model/pipeline flag such as `--disable_sph_mip`.
2. In the renderer, replace `pc.dir_encoding(...)` with a fixed direction encoding or zero tensor of the same `pc.sph_dim` when the flag is enabled.
3. Keep tensor shapes identical so `light_mlp` input dimensions remain unchanged.
4. Evaluate with `export_pbr_views.py --return_components` and compare `pbr_rgb`, `render`, `specular`, `diffuse`, and roughness statistics.

## no-factorization

Current status: not implemented as a CLI flag.

Relevant code:

- `gaussian_renderer/__init__.py`: specular is generated by `light_mlp(Sph-Mip direction, learned per-Gaussian feature)` and diffuse is `albedo_map`.
- `train.py`, `train-NeRF.py`, `train-NeRO.py`, `train-real.py`: losses consume `pbr_rgb` and sometimes stock `render`.

Minimal implementation path:

1. Add an explicit flag such as `--disable_ref_factorization`.
2. In renderer functions, bypass `spec_light + diff_light` and return either stock `render` or diffuse-only `linear2srgb(albedo_map)`.
3. Keep default behavior unchanged.
4. Export component buffers and report missing/disabled specular as an intentional ablation, not a renderer failure.

## reduced capacity

Current status: partially supported by existing config.

Safe settings:

- `--run_dim 64` versus `--run_dim 256`.
- `--roughness_lr 0` versus `--roughness_lr 0.002`.
- `--encoding_lr 0` to freeze Sph-Mip encoding weights after initialization.
- `--mlp_lr 0` to freeze the light MLP after initialization.

Remaining caution: freezing a module at random initialization tests optimization/capacity sensitivity, not a clean architectural removal.

## Evaluation entry after ablation

Use the round3 exporter/evaluator:

```bash
python experiments/ref_gs_limitation_analysis/export_pbr_views.py \
  --source_path "$SCENE" \
  --model_path "$MODEL" \
  --checkpoint "$MODEL/chkpnt31000.pth" \
  --split test \
  --max_views 64 \
  --return_components \
  --render_func auto \
  --out_dir "$EXPORT_DIR"

python experiments/ref_gs_limitation_analysis/evaluate_pbr.py \
  --export_dir "$EXPORT_DIR" \
  --out "$METRIC_DIR"
```

Do not report no-Sph-Mip or no-factorization as supported until the flag and renderer branch are implemented and verified.
