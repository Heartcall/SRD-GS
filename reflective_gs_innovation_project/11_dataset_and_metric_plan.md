## Dataset Selection Rationale
Derived Analysis:
- Primary proof should use synthetic reflective data where GT mesh/material can be verified or generated.
- Real scenes should be used for robustness and qualitative failure cases, but not overclaimed without GT.
- DTU/Tanks/BlendedMVS can show geometry generality but are not sufficient for reflective material claims.

Recommended minimal set:
1. Shiny Blender / Ref-NeRF scenes used by Ref-GS.
2. Glossy Synthetic / NeRO scenes used by Ref-GS.
3. One generic geometry benchmark subset such as DTU.
4. One real reflective subset for qualitative generalization.
5. Optional custom Blender glossy assets for full GT material/relighting.

## Required Ground Truth
- Mesh or dense point cloud for Chamfer/F-score.
- Normal maps or mesh-derived normals for normal MAE.
- Reflective/specular masks for region metrics.
- Albedo/roughness/material maps for material error.
- Novel-light renderings for relighting metrics.

Needs Verification: not all candidate datasets provide all GT fields. Missing GT must be reported as `unknown` and replaced only by clearly labeled proxy metrics.

## Metrics
Rendering:
- PSNR, SSIM, LPIPS.
- Refl-PSNR, Refl-SSIM, Refl-LPIPS.

Geometry:
- Chamfer Distance.
- F-score at multiple thresholds.
- normal MAE.
- depth error.
- normal-depth consistency.
- floating Gaussian count near surface.

Mesh:
- completeness.
- accuracy.
- connected component / hole count.
- surface noise.
- reflective-region qualitative overlays.

Texture / material:
- albedo error if GT exists.
- roughness error if GT exists.
- highlight leakage score.
- material consistency across views.
- relighting PSNR / LPIPS if novel-light GT exists.

Robustness:
- sparse-view setting.
- real-world scenes.
- runtime/memory/FPS.

## Reflective-region Protocol
1. Prefer dataset-provided reflective/specular masks.
2. If absent, estimate masks using multi-view color variance, residual `S_reflection`, roughness/specular weight and image intensity cues.
3. Validate masks visually on a subset.
4. Report mask generation method and include mask quality caveat.
5. Use masks for both rendering and geometry/material metrics where projection is valid.

## Texture / Material Protocol
1. Export mesh and UV maps.
2. Bake baseline direct RGB texture and proposed specular-free albedo.
3. Compute highlight leakage as high-intensity residual aligned with specular masks or view-dependent variance.
4. Render maps under novel light if GT or controlled lighting exists.
5. Do not claim roughness/metallic correctness without GT, relighting, or strong proxy evidence.

## Download and Preprocessing Notes
- Ref-GS README lists Shiny Blender Synthetic/Real, Glossy Synthetic and NeRF Synthetic; follow its folder layout for baseline comparability.
- Glossy Synthetic may require `nero2blender.py` conversion per Ref-GS README.
- For GT mesh use, document whether GT came from dataset assets, generated Blender mesh, or scan benchmark.
- For real scenes, document whether evaluation is proxy-only.

## Risks
- Dataset GT may be incomplete or hard to align.
- Reflective masks may be noisy.
- Material GT may be unavailable in public scenes.
- Real scenes may not support quantitative geometry/material claims.
- Custom Blender dataset creation may become a separate contribution.
